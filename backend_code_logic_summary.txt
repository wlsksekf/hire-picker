### 1. 리뷰 기능 (Review Feature)

**관련 파일:**
*   `backend/src/main/java/com/hirepicker/controller/ReviewController.java`
*   `backend/src/main/java/com/hirepicker/service/ReviewService.java`
*   `backend/src/main/java/com/hirepicker/repository/ComReviewRepository.java`
*   `backend/src/main/java/com/hirepicker/entity/ComReview.java`
*   `backend/src/main/java/com/hirepicker/dto/ReviewResponseDto.java`
*   `backend/src/main/java/com/hirepicker/dto/ReviewRequest.java`

**흐름 및 로직:**
1.  **`ReviewController.java`:**
    *   `@RequestMapping("/api/reviews")`로 리뷰 관련 API 엔드포인트를 정의합니다.
    *   **`GET /api/reviews/companies/{companyId}`:** 특정 `companyId`에 해당하는 모든 리뷰 목록을 `ReviewResponseDto` 형태로 반환합니다. `ReviewService`의 `getReviewsByCompany`를 호출합니다.
    *   **`GET /api/reviews/companies`:** 현재 로그인한 사용자(`CustomUserDetails`를 통해 인증)가 리뷰를 작성할 수 있는 회사 목록을 `CompanyReviewDto` 형태로 반환합니다. `ReviewService`의 `getReviewableCompanies`를 호출합니다.
    *   **`GET /api/reviews/{companyId}/my-review`:** 현재 로그인한 사용자가 특정 `companyId`에 대해 작성한 리뷰를 `ReviewRequest` 형태로 반환합니다. `ReviewService`의 `getMyReview`를 호출합니다. 리뷰가 없으면 404 Not Found를 반환합니다.
    *   **`POST /api/reviews/{companyId}`:** 특정 `companyId`에 대한 리뷰를 저장하거나 업데이트합니다. `ReviewRequest` DTO를 통해 리뷰 내용, 작성자 유형, 리뷰 인덱스(업데이트 시)를 받습니다. `ReviewService`의 `saveReview`를 호출합니다.
2.  **`ReviewService.java`:**
    *   `ComReviewRepository`, `CompanyRepository`, `PersonalUserRepository`, `ApplicationsRepository`, `JobPostingRepository`, `ResumeRepository`를 주입받아 사용합니다.
    *   **`getReviewsByCompany(Long companyId)`:** `ComReviewRepository`를 통해 `companyId`에 해당하는 모든 `ComReview` 엔티티를 조회하고, 이를 `ReviewResponseDto` 리스트로 변환하여 반환합니다.
    *   **`getReviewableCompanies(Long userId)`:**
        *   `ResumeRepository`를 통해 `userId`에 해당하는 이력서 목록을 조회합니다.
        *   `ApplicationsRepository`를 통해 이력서 ID와 상태("3" - 지원 완료/합격 등)를 기준으로 채용 공고 ID 목록을 조회합니다.
        *   `JobPostingRepository`를 통해 해당 채용 공고에 연결된 `JobPosting` 엔티티를 조회합니다.
        *   `JobPosting`에서 `Company` 정보를 추출하여 중복을 제거하고 `CompanyReviewDto` 리스트로 변환하여 반환합니다.
    *   **`getMyReview(Long companyId, PersonalUser personalUser)`:** `CompanyRepository`를 통해 `companyId`로 `Company` 엔티티를 찾고, `ComReviewRepository`를 통해 해당 회사와 `personalUser`가 작성한 리뷰를 조회하여 `Optional<ComReview>` 형태로 반환합니다.
    *   **`saveReview(Long companyId, String reviewContent, String reviewerType, PersonalUser personalUser, Long reviewIdx)`:**
        *   `companyId`로 `Company` 엔티티를 조회합니다.
        *   `reviewIdx`가 존재하면 기존 리뷰를 찾아 업데이트하고, 없으면 새로운 `ComReview` 엔티티를 생성하여 저장합니다.
        *   `PersonalUser` 객체를 직접 받아 리뷰 작성자를 설정합니다.
3.  **`ComReviewRepository.java`:**
    *   `JpaRepository`를 상속받아 `ComReview` 엔티티에 대한 CRUD 작업을 수행합니다.
    *   `findByCompanyAndPersonalUser(Company company, PersonalUser personalUser)`: 특정 회사와 개인 사용자에 대한 리뷰를 조회합니다.
    *   `findByCompanyCompanyIdx(Long companyId)`: 특정 `companyId`에 해당하는 모든 리뷰를 조회합니다.
4.  **`ComReview.java` (Entity):**
    *   `com_review` 테이블과 매핑되는 JPA 엔티티입니다.
    *   `reviewIdx` (PK), `content`, `reviewerType`, `company` (ManyToOne), `personalUser` (ManyToOne), `writeDate` 필드를 가집니다.
5.  **`ReviewResponseDto.java` (DTO):**
    *   프론트엔드로 리뷰 목록을 반환할 때 사용되는 DTO입니다. `content`, `reviewerType`, `writeDate` 필드를 가집니다.
6.  **`ReviewRequest.java` (DTO):**
    *   프론트엔드에서 리뷰를 저장/업데이트할 때 요청 본문으로 사용되는 DTO입니다. `review` (내용), `reviewerType`, `reviewIdx` 필드를 가집니다.

**백엔드 데이터 흐름:**
*   프론트엔드에서 리뷰 관련 요청이 오면 `ReviewController`가 이를 받아 `ReviewService`의 적절한 메서드를 호출합니다.
*   `ReviewService`는 `ComReviewRepository`, `CompanyRepository` 등 필요한 리포지토리를 통해 데이터베이스와 상호작용하여 데이터를 조회하거나 저장합니다.
*   조회된 엔티티는 DTO로 변환되어 `ReviewController`를 통해 프론트엔드로 응답됩니다.

### 2. 회사 목록 및 상세 정보 기능 (Company List & Details Feature)

**관련 파일:**
*   `backend/src/main/java/com/hirepicker/controller/CompanyApiController.java`
*   `backend/src/main/java/com/hirepicker/service/CompanyService.java`
*   `backend/src/main/java/com/hirepicker/repository/CompanyRepository.java`
*   `backend/src/main/java/com/hirepicker/entity/Company.java`
*   `backend/src/main/java/com/hirepicker/dto/CompanyDto.java`
*   `backend/src/main/java/com/hirepicker/dto/CompanySearchResponseDto.java`

**흐름 및 로직:**
1.  **`CompanyApiController.java`:**
    *   `@RequestMapping("/api")`로 회사 관련 API 엔드포인트를 정의합니다.
    *   **`GET /api/companies`:** 페이지네이션 및 검색(`query` 파라미터)을 적용하여 회사 목록을 `CompanyDto` 형태로 반환합니다. `employmentData.getCompanies`를 호출합니다. (참고: `employmentData`는 `EmploymentDataService` 또는 `EmploymentDataImpl`의 인스턴스로 보입니다.)
    *   **`GET /api/companies/{idx}`:** `companyIdx`를 이용하여 특정 회사의 상세 정보를 `CompanyDto` 형태로 반환합니다. `employmentData.getCompany`를 호출합니다.
    *   **`GET /api/companies/search?name={name}`:** 회사 이름(`name` 파라미터)으로 회사를 검색하고 `CompanySearchResponseDto` 리스트를 반환합니다. `companyService.searchByName`을 호출합니다.
    *   **`POST /api/CreateCompanies`:** 새로운 회사 정보를 등록합니다. `CompanyDto`를 요청 본문으로 받아 `companyService.createCompany`를 호출합니다.
2.  **`CompanyService.java`:**
    *   `CompanyRepository`를 주입받아 사용합니다.
    *   **`searchByName(String name)`:** `CompanyRepository`의 `findByCompanyNameContainingIgnoreCase`를 사용하여 회사 이름으로 검색하고, 결과를 `CompanySearchResponseDto` 리스트로 변환하여 반환합니다. 자동 완성을 위해 상위 10개 결과만 가져옵니다.
    *   **`createCompany(CompanyDto dto)`:**
        *   `companyRepository`를 통해 회사 이름 및 사업자 등록 번호 중복을 검사합니다.
        *   `CompanyDto`를 기반으로 `Company` 엔티티를 생성하고, `status`를 "PENDING"으로, `regDate`를 현재 날짜로 설정한 후 저장합니다.
        *   저장된 `Company` 엔티티를 `CompanyDto`로 변환하여 반환합니다.
    *   **`isCompanyNameDuplicate(String companyName)`:** `companyRepository.existsByCompanyName`를 호출하여 회사 이름 중복 여부를 확인합니다.
    *   **`isBusinessNumberDuplicate(String businessNumber)`:** `companyRepository.existsByBusinessNumber`를 호출하여 사업자 등록 번호 중복 여부를 확인합니다.
3.  **`CompanyRepository.java`:**
    *   `JpaRepository`를 상속받아 `Company` 엔티티에 대한 CRUD 작업을 수행합니다.
    *   `findByCompanyName`, `findAllByCompanyName`, `findByCompanyId`, `findByCompanyIdx`, `findByCompanyNameContainingIgnoreCase`, `findByCorpCode` 등 다양한 조회 메서드를 제공합니다.
    *   `existsByCompanyName`, `existsByBusinessNumber`: 중복 확인을 위한 메서드입니다.
4.  **`Company.java` (Entity):**
    *   `company` 테이블과 매핑되는 JPA 엔티티입니다.
    *   `companyIdx` (PK), `companyId`, `companyName`, `description`, `websiteUrl`, `businessNumber`, `logoUrl`, `companyType`, `ceoName`, `address`, `employeeCount`, `corpCode`, `status`, `regDate`, `salesAmount`, `welfareBenefits`, `imgPath` 필드를 가집니다.
5.  **`CompanyDto.java` (DTO):**
    *   회사 정보를 프론트엔드로 전달하거나 요청 본문으로 받을 때 사용되는 레코드 타입 DTO입니다.
6.  **`CompanySearchResponseDto.java` (DTO):**
    *   회사 검색 결과(자동 완성 등)를 반환할 때 사용되는 DTO입니다. `companyIdx`와 `companyName` 필드를 가집니다.

**백엔드 데이터 흐름:**
*   프론트엔드에서 회사 목록 조회, 상세 정보 조회, 검색, 등록 요청이 오면 `CompanyApiController`가 이를 받아 `CompanyService` 또는 `EmploymentData`의 적절한 메서드를 호출합니다.
*   `CompanyService`는 `CompanyRepository`를 통해 데이터베이스와 상호작용하여 데이터를 조회, 저장, 중복 확인합니다.
*   조회되거나 저장된 엔티티는 DTO로 변환되어 `CompanyApiController`를 통해 프론트엔드로 응답됩니다.

### 3. 관심 기업 기능 (Company Alarm Feature)

**관련 파일:**
*   `backend/src/main/java/com/hirepicker/controller/CompanyalarmsController.java`
*   `backend/src/main/java/com/hirepicker/service/CompanyalarmsService.java`
*   `backend/src/main/java/com/hirepicker/repository/CompanyalarmsRepository.java`
*   `backend/src/main/java/com/hirepicker/entity/Companyalarms.java`
*   `backend/src/main/java/com/hirepicker/dto/CompanyalarmsResponseDto.java`

**흐름 및 로직:**
1.  **`CompanyalarmsController.java`:**
    *   `@RequestMapping("/api/company-alarms")`로 관심 기업 관련 API 엔드포인트를 정의합니다.
    *   **`POST /api/company-alarms`:** 로그인한 사용자(`CustomUserDetails`를 통해 `pUserIdx` 획득)가 특정 `companyIdx`를 관심 기업으로 등록합니다. `CompanyalarmsService`의 `addCompanyAlarm`을 호출하고, 성공 시 `CompanyalarmsResponseDto`를 반환합니다.
    *   **`DELETE /api/company-alarms/{pUserIdx}/{companyIdx}`:** 로그인한 사용자가 특정 `companyIdx`를 관심 기업 목록에서 제거합니다. `CompanyalarmsService`의 `removeCompanyAlarm`을 호출합니다.
    *   **`GET /api/company-alarms/user/{pUserIdx}`:** 로그인한 사용자의 `pUserIdx`에 해당하는 모든 관심 기업의 `companyIdx` 목록을 반환합니다. `CompanyalarmsService`의 `getLikedCompanyIdsByPersonalUser`를 호출합니다.
    *   **`GET /api/company-alarms/idx-by-email?email={email}`:** 이메일을 통해 개인 사용자의 `pUserIdx`를 조회합니다. `AuthService`의 `findPersonalUserIdxByEmail`을 호출합니다.
2.  **`CompanyalarmsService.java`:**
    *   `CompanyalarmsRepository`, `PersonalUserRepository`, `CompanyRepository`를 주입받아 사용합니다.
    *   **`addCompanyAlarm(Long pUserIdx, Long companyIdx)`:**
        *   `PersonalUserRepository`와 `CompanyRepository`를 통해 `PersonalUser` 및 `Company` 엔티티를 조회합니다.
        *   `CompanyalarmsRepository`를 통해 이미 관심 기업으로 등록되어 있는지 확인합니다.
        *   새로운 `Companyalarms` 엔티티를 생성하여 저장합니다.
    *   **`removeCompanyAlarm(Long pUserIdx, Long companyIdx)`:** `CompanyalarmsRepository`를 통해 특정 `pUserIdx`와 `companyIdx`에 해당하는 관심 기업을 삭제합니다.
    *   **`getLikedCompanyIdsByPersonalUser(Long pUserIdx)`:** `CompanyalarmsRepository`를 통해 `pUserIdx`에 해당하는 모든 `Companyalarms` 엔티티를 조회하고, 각 엔티티에서 `companyIdx`를 추출하여 리스트로 반환합니다.
    *   **`isCompanyLikedByUser(Long pUserIdx, Long companyIdx)`:** 특정 사용자가 특정 회사를 관심 기업으로 등록했는지 여부를 확인합니다.
3.  **`CompanyalarmsRepository.java`:**
    *   `JpaRepository`를 상속받아 `Companyalarms` 엔티티에 대한 CRUD 작업을 수행합니다. 복합 키(`CompanyalarmsId`)를 사용합니다.
    *   `findByPersonalUserId_Id`: 특정 개인 사용자의 모든 관심 기업을 조회합니다.
    *   `findByPersonalUserId_IdAndCompanyIdx_CompanyIdx`: 특정 개인 사용자와 회사에 대한 관심 기업을 조회합니다.
    *   `deleteByPersonalUserId_IdAndCompanyIdx_CompanyIdx`: 특정 개인 사용자와 회사에 대한 관심 기업을 삭제합니다.
4.  **`Companyalarms.java` (Entity):**
    *   `company_alarm` 테이블과 매핑되는 JPA 엔티티입니다.
    *   `PersonalUser` (ManyToOne, PK의 일부), `Company` (ManyToOne, PK의 일부) 필드를 가집니다. `CompanyalarmsId` 복합 키를 사용합니다.
5.  **`CompanyalarmsResponseDto.java` (DTO):**
    *   관심 기업 등록/조회 응답 시 사용되는 DTO입니다. `pUserIdx`와 `companyIdx` 필드를 가집니다.

**백엔드 데이터 흐름:**
*   프론트엔드에서 관심 기업 등록/해제/조회 요청이 오면 `CompanyalarmsController`가 이를 받아 `CompanyalarmsService`의 적절한 메서드를 호출합니다.
*   `CompanyalarmsService`는 `CompanyalarmsRepository`, `PersonalUserRepository`, `CompanyRepository`를 통해 데이터베이스와 상호작용합니다.
*   처리 결과는 DTO로 변환되어 `CompanyalarmsController`를 통해 프론트엔드로 응답됩니다.

### 4. 캘린더 기능 (Calendar Feature)

**관련 파일:**
*   `backend/src/main/java/com/hirepicker/controller/CompanyApiController.java` (캘린더 관련 엔드포인트 포함)
*   `backend/src/main/java/com/hirepicker/service/EmploymentDataService.java`
*   `backend/src/main/java/com/hirepicker/repository/JobPostingRepository.java`
*   `backend/src/main/java/com/hirepicker/repository/EmpEventRepository.java`
*   `backend/src/main/java/com/hirepicker/entity/JobPosting.java`
*   `backend/src/main/java/com/hirepicker/entity/EmpEvent.java`
*   `backend/src/main/java/com/hirepicker/dto/CalendarJobPostingDto.java`
*   `backend/src/main/java/com/hirepicker/dto/CalendarEmpEventDto.java`

**흐름 및 로직:**
1.  **`CompanyApiController.java`:**
    *   **`GET /api/calendar/job-postings`:** 캘린더에 표시할 모든 채용 공고 목록을 `CalendarJobPostingDto` 형태로 반환합니다. `employmentData.getAllJobPostingsForCalendar()`를 호출합니다.
    *   **`GET /api/calendar/emp-events?regions={regions}`:** 캘린더에 표시할 채용 행사 목록을 `CalendarEmpEventDto` 형태로 반환합니다. `regions` 파라미터로 필터링할 수 있습니다. `employmentData.getAllEmpEventsForCalendarByRegions()`를 호출합니다.
2.  **`EmploymentDataService.java`:** (이름은 `Service`지만, 실제로는 외부 데이터 동기화 및 가공 로직을 많이 포함하고 있습니다. `EmploymentData` 인터페이스의 구현체로 보입니다.)
    *   `JobPostingRepository`, `EmpEventRepository` 등을 주입받아 사용합니다.
    *   **`getAllJobPostingsForCalendar()`:** `JobPostingRepository`에서 모든 `JobPosting` 엔티티를 조회하고, 이를 `CalendarJobPostingDto` 리스트로 변환하여 반환합니다.
    *   **`getAllEmpEventsForCalendarByRegions(List<String> regions)`:** `EmpEventRepository`에서 모든 `EmpEvent` 엔티티를 조회하거나 `regions`에 따라 필터링하여 `CalendarEmpEventDto` 리스트로 변환하여 반환합니다.
    *   **데이터 동기화 로직:** `synchronizePublicJobs()`, `synchronizeEvents()`, `synchronizeCompanies()`, `SyncDartInfo()` 등 외부 API(Work24, DART)로부터 데이터를 가져와 DB에 저장하는 복잡한 동기화 로직을 포함합니다. 이 부분은 캘린더 기능 자체보다는 데이터 수집 및 관리에 더 가깝습니다.
3.  **`JobPostingRepository.java`:**
    *   `JpaRepository`를 상속받아 `JobPosting` 엔티티에 대한 CRUD 작업을 수행합니다.
    *   `findByPostingId`, `findByTitleContaining`, `findByTitleContainingAndLocation`, `findByPostingIdxIn` 등 조회 메서드를 제공합니다.
4.  **`EmpEventRepository.java`:**
    *   `JpaRepository`를 상속받아 `EmpEvent` 엔티티에 대한 CRUD 작업을 수행합니다.
    *   `findByEventCode` 등 조회 메서드를 제공합니다. `EmpEventRepositoryCustom` 인터페이스를 통해 커스텀 쿼리도 지원합니다.
5.  **`JobPosting.java` (Entity):**
    *   `job_posting` 테이블과 매핑되는 JPA 엔티티입니다.
    *   `postingIdx` (PK), `postingId`, `company` (ManyToOne), `cUserIdx`, `title`, `employmentType`, `location`, `experience_level`, `jobType`, `startDate`, `endDate` 필드를 가집니다.
6.  **`EmpEvent.java` (Entity):**
    *   `emp_event` 테이블과 매핑되는 JPA 엔티티입니다.
    *   `eventIdx` (PK), `eventCode`, `eventName`, `eventDuration`, `area` 필드를 가집니다.
7.  **`CalendarJobPostingDto.java` (DTO):**
    *   캘린더에 표시할 채용 공고 정보를 담는 DTO입니다. `id`, `companyName`, `start`, `end`, `allDay`, `type` 필드를 가집니다. `JobPosting` 엔티티로부터 변환하는 `fromEntity` 팩토리 메서드를 제공합니다.
8.  **`CalendarEmpEventDto.java` (DTO):**
    *   캘린더에 표시할 채용 행사 정보를 담는 DTO입니다. `id`, `title`, `start`, `end`, `allDay`, `type`, `area` 필드를 가집니다. `EmpEvent` 엔티티로부터 변환하는 `fromEntity` 팩토리 메서드를 제공합니다.

**백엔드 데이터 흐름:**
*   프론트엔드에서 캘린더 데이터 요청이 오면 `CompanyApiController`가 이를 받아 `EmploymentDataService`의 캘린더 관련 조회 메서드를 호출합니다.
*   `EmploymentDataService`는 `JobPostingRepository`와 `EmpEventRepository`를 통해 데이터베이스에서 `JobPosting` 및 `EmpEvent` 엔티티를 조회합니다.
*   조회된 엔티티는 `CalendarJobPostingDto` 또는 `CalendarEmpEventDto`로 변환되어 `CompanyApiController`를 통해 프론트엔트로 응답됩니다.
*   데이터 동기화는 스케줄링된 작업(`@Scheduled`) 또는 수동 트리거 API를 통해 `EmploymentDataService`에서 처리됩니다.

### 5. 기업 회원가입 기능 (Company Signup Feature)

**관련 파일:**
*   `backend/src/main/java/com/hirepicker/controller/AuthController.java`
*   `backend/src/main/java/com/hirepicker/service/AuthService.java`
*   `backend/src/main/java/com/hirepicker/repository/CompanyUserRepository.java`
*   `backend/src/main/java/com/hirepicker/entity/CompanyUser.java`
*   `backend/src/main/java/com/hirepicker/dto/CompanySignupRequestDto.java`
*   `backend/src/main/java/com/hirepicker/repository/CompanyRepository.java`
*   `backend/src/main/java/com/hirepicker/entity/Company.java`

**흐름 및 로직:**
1.  **`AuthController.java`:**
    *   `@RequestMapping("/api/auth")`로 인증 관련 API 엔드포인트를 정의합니다.
    *   **`POST /api/auth/signup/company`:** 기업 회원가입 요청을 처리합니다. `CompanySignupRequestDto` (JSON 데이터)와 `MultipartFile` (인증 파일)을 `@RequestPart`로 받습니다.
        *   `EmailService`를 통해 이메일 인증 상태를 검증합니다.
        *   `AuthService`의 `registerCompanyUserWithDocument`를 호출하여 실제 회원가입 로직을 수행합니다.
        *   성공 시 "회원가입 신청이 완료되었습니다. 관리자 승인 후 로그인 가능합니다." 메시지를 반환합니다.
    *   **`POST /api/auth/send-verification`:** 회원가입을 위한 이메일 인증 코드를 발송합니다. `AuthService.isEmailDuplicate`로 이메일 중복을 확인하고 `EmailService.sendVerificationCode`를 호출합니다.
    *   **`POST /api/auth/check-verification`:** 이메일 인증 코드를 확인합니다. `EmailService.verifyCodeAndSetStatus`를 호출합니다.
2.  **`AuthService.java`:**
    *   `AuthenticationManager`, `PersonalUserRepository`, `CompanyUserRepository`, `CompanyRepository`, `RefreshTokenRepository`, `JwtTokenProvider`, `PasswordEncoder`, `Environment`, `S3UploadService`를 주입받아 사용합니다.
    *   **`registerCompanyUserWithDocument(CompanySignupRequestDto request, MultipartFile file, HttpServletResponse response)`:**
        *   **중복 체크:** `companyUserRepository.existsByLoginId` 및 `isEmailDuplicate`를 통해 아이디와 이메일 중복을 확인합니다.
        *   **회사 정보 조회:** `companyRepository`를 통해 `request.getCompanyIdx()`에 해당하는 `Company` 엔티티를 조회합니다.
        *   **파일 검증:** `validateVerificationFile` 메서드를 호출하여 파일 크기 및 확장자를 검증합니다.
        *   **S3 파일 업로드:** `s3UploadService.uploadFile`을 사용하여 인증 파일을 S3에 업로드하고 파일 URL을 받습니다.
        *   **`CompanyUser` 생성 및 저장:** `CompanyUser.builder()`를 사용하여 `CompanyUser` 엔티티를 생성합니다. 이때 `verificationFile` 필드에 S3 URL을 저장하고, `isApproved` 상태를 `ApprovalStatus.PENDING`으로 설정합니다. `companyUserRepository.save`를 통해 저장합니다.
        *   **주의:** 기업 회원은 관리자 승인 전까지 로그인할 수 없으므로, 이 메서드에서는 토큰 발급 및 쿠키 설정 로직을 수행하지 않습니다.
    *   **`isEmailDuplicate(String email)`:** `personalUserRepository.existsByEmail` 또는 `companyUserRepository.existsByEmail`을 통해 개인/기업 사용자 이메일 중복 여부를 확인합니다.
    *   **`validateVerificationFile(MultipartFile file)`:** 파일 크기(최대 10MB) 및 허용 확장자(pdf, jpg, jpeg, png)를 검증합니다.
3.  **`CompanyUserRepository.java`:**
    *   `JpaRepository`를 상속받아 `CompanyUser` 엔티티에 대한 CRUD 작업을 수행합니다.
    *   `findByLoginId`, `findByEmail`, `existsByEmail`, `existsByLoginId` 등 조회 및 중복 확인 메서드를 제공합니다.
4.  **`CompanyUser.java` (Entity):**
    *   기업 사용자 정보를 담는 JPA 엔티티입니다.
    *   `loginId`, `password`, `name`, `email`, `phoneNumber`, `company` (ManyToOne), `verificationFile`, `isApproved` 필드를 가집니다.
5.  **`CompanySignupRequestDto.java` (DTO):**
    *   기업 회원가입 요청 시 사용되는 DTO입니다. `id`, `password`, `name`, `email`, `phone_number`, `verificationCode`, `companyIdx` 필드를 가집니다. `@NotBlank`, `@Email`, `@NotNull` 등의 유효성 검사 어노테이션이 적용되어 있습니다.
6.  **`CompanyRepository.java` (참고):**
    *   `AuthService`에서 `Company` 엔티티를 조회하는 데 사용됩니다.
7.  **`Company.java` (Entity, 참고):**
    *   `CompanyUser` 엔티티와 ManyToOne 관계로 연결됩니다.

**백엔드 데이터 흐름:**
1.  프론트엔드에서 기업 회원가입 요청이 오면 `AuthController`가 이를 받아 `AuthService`의 `registerCompanyUserWithDocument`를 호출합니다.
2.  `AuthService`는 이메일 인증 상태 확인, 아이디/이메일 중복 확인, `Company` 엔티티 조회, 인증 파일 검증 및 S3 업로드를 수행합니다.
3.  모든 검증 및 처리 후 `CompanyUser` 엔티티를 `PENDING` 상태로 `CompanyUserRepository`를 통해 데이터베이스에 저장합니다.
4.  성공/실패 메시지가 `AuthController`를 통해 프론트엔트로 응답됩니다.
