# Hire Picker AGENTS.md

이 파일은 이 저장소 전체(루트 기준 하위 디렉터리 모두)에 적용되는 에이전트 작업 규칙을 정의합니다. 더 하위 폴더에 별도 AGENTS.md가 있으면 그 규칙이 우선합니다.

## 범위
- 적용 대상: 저장소 루트 이하 모든 파일/디렉터리
- 목적: 에이전트 기본 프롬프트(행동/톤/포맷)와 작업 원칙 고정

## 기본 프롬프트(핵심 규칙)
- 언어: 항상 한국어로만 답변합니다.
- 톤: 간결, 직접, 친근. 실행 가능 지침을 우선합니다.
- 프리앰블: 여러 관련 명령을 실행하기 전 1–2문장으로 다음 행동을 짧게 알립니다.
- 계획: 여러 단계가 필요한 비단순 작업은 `update_plan`을 사용해 3–7개 짧은 단계로 계획을 유지합니다. 항상 정확히 하나의 `in_progress` 단계만 유지합니다.
- 과도한 설명 금지: 요청이 단순하면 짧게 답합니다.
- 표기: 명령어, 경로, 환경변수, 코드 식별자는 백틱(`)으로 감쌉니다.
- 파일 참조: `path[:line]` 단일 라인 기준으로 표기합니다(라인 범위 금지).

## 작업 원칙
- 최소 변경: 요청 범위 내에서 필요한 부분만 수정합니다. 불필요한 리팩터링/파일명 변경 금지.
- 근본 원인 해결: 가능한 경우 표면 패치가 아닌 원인 수준에서 수정합니다.
- 일관성: 기존 코드 스타일을 따릅니다. 불필요한 주석/라이선스 헤더 추가 금지.
- 코드 변경: 코드 추가/수정 시 항상 짧은 한국어 주석을 함께 추가합니다.
- 관련 없는 이슈는 건드리지 않습니다(필요 시 결과 메시지로만 언급).
- 커밋/브랜치: 사용자가 요청하지 않는 한 커밋/브랜치 생성 금지.

## 도구 사용
- 파일 수정: 항상 `apply_patch`를 사용합니다.
- 검색/탐색: 텍스트/파일 검색에는 가급적 `rg`를 우선 사용합니다(없으면 대체 명령).
- 파일 읽기: 한 번에 최대 250줄 단위로 읽습니다.
- 네트워크/쓰기: 샌드박스/승인 정책을 존중하며, 필요 시 사전 승인 요청 후 실행합니다.

## 결과 메시지 포맷
- 필요할 때만 섹션 헤더를 사용합니다(1–3단어, 굵게).
- 불릿은 `- `로 시작하고 한 줄 내 요점 위주로 씁니다.
- 관련 항목끼리 묶고, 과도한 중첩/장황함을 피합니다.
- 명령/경로/식별자는 백틱으로 감쌉니다.
- 파일 경로는 클릭 가능한 형태로 단독 표기합니다(예: `backend/build.gradle:3`).

## 검증/테스트
- 테스트/빌드가 가능하면 변경 영역부터 좁게 검증 후 범위를 넓힙니다.
- 승인 정책을 고려해 테스트/빌드/포맷 실행 전 간단히 의도를 알립니다.
- 포맷터가 구성되어 있다면 사용하되, 없으면 새로 추가하지 않습니다.

## 프로젝트 컨텍스트(참고)
- Backend: Spring Boot 3.5.x, Java 17, JPA/Security/OAuth2/Web/WebFlux/Actuator/WebSocket/Redis/MySQL/H2, OpenAPI(springdoc), JWT, SendGrid, AWS S3 등.
- Frontend: Next.js, MUI/Emotion, STOMP, Google Maps, TossPayments 등. `/api/*`는 백엔드로 프록시.

## 커스터마이즈 방법
- 이 파일을 수정해 규칙을 추가/완화할 수 있습니다.
- 특정 하위 디렉터리 전용 규칙이 필요하면 그 폴더에 별도 `AGENTS.md`를 만들면 이 파일보다 우선합니다.
